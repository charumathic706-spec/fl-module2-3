How Blockchain Is Implemented Here
1. Split 2 writes per-round trust and model-hash logs into trust_training_log.json.
2. Governance builds a hash chain and appends immutable records in governance_report.json and hash_chain.json.
3. Backend gateway commits each round to selected ledger backend: simulation, ganache, or fabric.
4. Each round contains a commit tx and audit tx, visible below as the on-chain style timeline.
5. Integrity is verified by comparing round-to-round prev_block_hash linkage and recorded tamper alerts.
